ЗАДАНИЕ
Сделать контроллеры для url-ов и html страницы для сайта jobboard.
Структура сайта (в формате url и что должно быть на странице):
/ - index page (navbar и таблица с вакансиями)

/vacancy/add - navbar и форма для добавления вакансии

/company - navbar и список компаний

/company/add - navbar и форма для добавления компании

/profile - navbar и информация о пользователе (которая описана в ТЗ, которое я скидывал раньше). 
Можно отображать данные в любом формате (например с помощью тэгов <p> и <h1>)

/vacancy/{айди вакансии} - navbar и страница с описанием конкретной вакансии и кнопкой Apply.
При нажатии на кнопку apply должен происходить redirect на url /vacancy/{айди вакансии}/apply

/vacancy/{айди вакансии}/apply - navbar и форма для отправки данных на сервер. В эту форму 
пользователь должен вводить данные необходимые для создания отклика на вакансию (описание этих данных
есть в ТЗ).

/company/{айди компании} - страница с описанием компании и отображением отзывов о компании. 
Отображать в любом виде (дизайн страницы не важен, главное, чтобы информация воспринималась визуально).
Также на этой странице должна быть кнопка Create Review, которая делает redirect на url
/company/{айди компании}/review

/company/{айди компании}/review - navbar и форма для оставления отзыва на компанию (как описано в ТЗ).

Резюме: То есть основная задача реализовать только контроллеры и сверстать основные html-страницы с произвольным
дизайном. На данном этапе важно изучить средства Django для работы с сетью и обработки запросов и ответов: 
views, templates engine, tags, template inheritance, forms, middleware, files, media, etc. 

В качестве хранилища пока что достаточно использовать mock-хранилище (например которое я реализовал чисто в оперативной памяти)
или просто захардкодить данные в HTML. На следующих занятиях мы уже добавим базу данных к этому всему.
